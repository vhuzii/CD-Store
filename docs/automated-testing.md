# Автоматизоване тестування — CD Маркет

**Проєкт:** CD Маркет — маркетплейс CD-дисків в Україні  
**Тип документа:** Звіт з автоматизованого тестування (Automated Testing Report)  
**Інструменти:** Vitest (Test Runner), React Testing Library (DOM Testing), Zustand (State Management Test), jsdom (DOM Environment)  
**Середовище:** Node.js v18+, CI/CD Pipeline  
**Дата останнього тестування:** 2026-04-14  

---

## 1. Огляд стратегії тестування (Testing Strategy Overview)

Цей документ описує результати виконання автоматизованих тестів для фронтенд-частини проєкту. Увесь конвеєр із **60 сценаріїв** розбито на два типи згідно з пірамідою тестування:
1. **Інтеграційні / Компонентні тести (Integration Tests) — 30 тестів:** Перевірка роботи віртуального DOM, симуляція кліків користувача, перевірка роутингу.
2. **Модульні / Юніт-тести (Unit Tests) — 30 тестів:** Суворо ізольована перевірка чистих функцій, безпеки транзакцій стану (Zustand reducers) та консистентності статичних довідників без залучення React.

### 1.1. Тестове середовище та моки (Mocks & Setup)
- **DOM:** `jsdom` інтегрований з Vitest для повноцінної симуляції браузера.
- **Глобальні об'єкти:** Створено стаб для `window.matchMedia`, необхідний для UI-бібліотек.
- **Сторонні бібліотеки:** `framer-motion` (анімації відключені для швидкості візуалізації та стабілізації UI в тестах), `Supabase` (auth та onAuthStateChange замокано для повної ізоляції тесту від мережі).
- **Роутинг:** Використовується `MemoryRouter` для швидкої симуляції навігації.
- **Стан (State):** Перед кожним окремим тестом глобальний стор `useAppStore` жорстко скидається до початкових еталонних даних `mockData`.
