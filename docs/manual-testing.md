### BUG-001: Розрив між Supabase auth та Zustand currentUser

**Пріоритет:** Critical  
**Статус:** Open  
**Компоненти:** `src/hooks/useAuth.ts`, `src/lib/store.ts`, всі сторінки

**Опис:**  
Навбар (`Navbar.tsx`) використовує `useAuth()` (Supabase) для визначення авторизації, тоді як більшість сторінок (`CartPage`, `SellPage`, `CheckoutPage`, `MessagesPage`, `CDCard`, `CDDetailPage`) використовують `useAppStore().currentUser` (Zustand). Ці два стейти ніколи не синхронізуються.

**Кроки відтворення:**
1. Увійти через Google OAuth (Supabase auth)
2. Навбар показує авторизований стан (посилання на чат, продаж, кошик)
3. Відкрити `/cart` — показується "Увійдіть, щоб переглянути корзину"
4. Відкрити `/sell` — порожня сторінка (redirect)
5. Відкрити `/cd/cd1` — кнопки "Купити" та "Написати" не відображаються

**Очікуваний результат:** Після входу через Google всі сторінки мають розпізнавати авторизованого користувача.

**Фактичний результат:** Навбар показує авторизований стан, але решта сторінок вважають користувача неавторизованим.

**Рекомендація:** Синхронізувати Supabase auth з Zustand store або перевести всі сторінки на `useAuth()`.

---

### BUG-002: Сторінки /sell та /checkout показують порожню сторінку замість перенаправлення

**Пріоритет:** High  
**Статус:** Open  
**Компоненти:** `src/pages/SellPage.tsx`, `src/pages/CheckoutPage.tsx`

**Опис:**  
Сторінки `/sell` та `/checkout` викликають `navigate("/auth")` всередині тіла компонента (не в useEffect) і повертають `null`. Це призводить до порожньої сторінки — redirect не спрацьовує.

**Кроки відтворення:**
1. Відкрити сайт без авторизації
2. Перейти на `/sell`
3. Спостерігати порожню сторінку (тільки навбар)

**Очікуваний результат:** Автоматичне перенаправлення на `/auth`.

**Фактичний результат:** Порожня сторінка, URL залишається `/sell`.

**Рекомендація:** Використовувати `useEffect` для navigate або компонент `<Navigate to="/auth" replace />` з react-router-dom.
